Hexo 追番插件 — Bangumis & Bilibili-Bangumi

大家好～今天给大家推荐两款 Hexo 博客的追番页面插件，让你在博客上展示自己的追番列表 (ゝω·)ノ

免责声明：本文仅提供操作选项与配置参考，具体实践中如遇到任何错误或问题，均与本站无关。教学方法来源于站主个人经验总结及互联网公开内容，请根据自身实际情况谨慎操作。

一、为什么需要追番插件

作为一名动漫爱好者，追番是日常。如果你有自己的 Hexo 博客，把追番列表集成到博客中不仅能展示你的动漫品味，还能：

  • 自动同步追番进度，无需手动更新
  • 精美的卡片式展示封面、评分和简介
  • 访客可以快速了解你正在看的番剧
  • 结合 Bangumi / Bilibili 数据源，信息自动更新

二、插件一：hexo-bangumis

2.1 简介

hexo-bangumis 是一款为 Hexo 博客添加 Bangumi 追番页面的插件，由 mmdjiji 开发。

它使用 bgm.tv 的官方 API 来爬取追番数据，并在博客中生成精美的追番列表页面。

特色功能：

特性 说明
bgm.tv 官方 API 使用 Bangumi 官方数据源，番剧更多，可用性更高
数据缓存 支持缓存番剧数据，防止上游服务故障导致连锁效应
封面图本地化 支持下载番剧封面到本地，纯离线场景下也能正常部署
自定义番剧列表 极端条件下可手动编辑文件自定义番剧，添加 bgm.tv 上没有的番剧
离线部署 只要爬取过番剧数据，就算 bgm.tv 上删除了该番剧也能继续使用
GitHub Actions 自动更新 支持配置定时任务自动更新追番数据

三、插件二：hexo-bilibili-bangumi

3.1 简介

hexo-bilibili-bangumi 是一款功能更全面的 Hexo 追番/追剧/游戏页面插件，由 HCLonely 开发。它支持多个数据源，不仅可以展示追番列表，还能展示追剧和游戏列表。

特色功能：

特性 说明
多数据源支持 支持 Bilibili、Bangumi、AniList、Simkl 四种数据源
追番 + 追剧 + 游戏 三合一页面，功能最全面
图片懒加载 支持图片懒加载，提升页面性能
WebP 格式 支持 WebP 格式图片，大幅减小体积
进度条展示 支持展示追番进度
响应式设计 适配移动端和桌面端
图片镜像缓存 支持配置封面镜像缓存解决防盗链
手动添加数据 支持手动添加番剧数据

需要离线部署 hexo-bangumis
需要追剧 + 游戏页面 hexo-bilibili-bangumi
追求功能全面 hexo-bilibili-bangumi
